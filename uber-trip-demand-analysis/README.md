# Uber Trip Demand & Revenue Analysis Using SQL

## Problem Statement

Ride-sharing companies like Uber generate large volumes of trip data every day. 
Analyzing this data helps identify peak demand periods, revenue-driving vehicle categories, customer behavior, and operational improvement opportunities.

This project uses SQL to analyze Uber trip data and generate business insights related to trip demand, revenue trends, airport trips, vehicle performance, and ride patterns.

## Business Questions

- What are the peak demand hours for Uber trips?
- Which vehicle type generates the highest revenue?
- Which pickup locations generate the most trips and revenue?
- How do weekend trips compare with weekday trips?
- What percentage of trips are airport-related?
- Which time periods have the highest ride demand?

## Technologies Used

- MySQL
- SQL
- Data Analysis
- Window Functions
- Aggregation Functions
- Business Intelligence

## Data Cleaning & Preparation

- Imported CSV data into MySQL
- Removed duplicate trip records
- Applied ROW_NUMBER() window function for final deduplication
- Validated imported records
- Structured cleaned data for analysis

## Analysis Performed

- Trip demand analysis
- Revenue analysis
- Peak hour analysis
- Vehicle performance analysis
- Airport trip analysis
- Weekend vs weekday comparison
- Pickup location analysis

## Key Insights

- Peak ride demand occurred during high-traffic hours.
- Premium vehicle categories generated higher revenue.
- Airport-related trips contributed significantly to total bookings.
- Certain pickup locations consistently showed high customer demand.
- Weekend ride behavior showed variations in average fare patterns.

## Business Recommendations

- Increase driver availability during peak demand hours.
- Optimize airport ride operations for improved efficiency.
- Focus premium vehicle allocation in high-revenue locations.
- Apply demand-based pricing strategies during weekends and busy periods.
